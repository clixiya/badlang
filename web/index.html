Syntax Showcase

Expressive DSL, Predictable Runtime

Cover common API testing patterns with very little ceremony.

Assertions and Variables
expect status 200
expect json.user.id == 1
expect json.token exists
 
let token = json.token
let auth = bearer token
Templates and Imports
export template get_user {
  method GET
  path "/users/1"
}
 
import "shared.bad" only get_user
send req get_user
Control Flow and Retry
if status != 200 {
  stop_all because "health check failed"
}
 
send GET "/jobs" {
  retry 3
  retry_backoff exponential
}
Hooks and Timing
before_each { print "starting" }
after_each { print "done" }
 
time_start auth_flow
send POST "/auth/login"
time_stop auth_flow
expect time auth_flow < 2000
VS Code Extension

BAD Language Tools for VS Code

Syntax highlighting, snippets, diagnostics, run actions, settings panel, and icon theme.

Syntax Highlighting
TextMate grammar with rich keyword coverage for .bad and .badrc.
IntelliSense + Snippets
Fast completions for test blocks, requests, assertions, hooks, and runtime flags.
Run from Editor
Run current BAD file directly with command palette and sidebar actions.
Runtime Config Panel
Edit one .badrc with advanced history controls from a dedicated webview.

Install the Extension

1
Build the VSIX from extenstion/ or use the packaged release.
2
Install via CLI:
code --install-extension extenstion/dist/bad-language-tools-1.1.0.vsix
3
Open any .bad file and use BAD commands from the palette.
4
Configure settings with BAD: Settings and BAD: Runtime Config.
